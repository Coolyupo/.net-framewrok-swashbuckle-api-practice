API Object routes

API對象描述了Kong公開的API。當消費者從代理端口呼叫時，Kong需要知道如何檢索API。每個API對象必須指定的某種組合hosts，uris和methods。Kong將代理對API的所有請求到指定的上游URL。

Endpoint

POST /apis/

Request Body

屬性 描述
name API名稱。
hosts
半可選		 指向API的網域名稱以逗號分隔列表。例如：example.com。hosts，uris或methods至少一個需填才符合規定。
uris
半可選		 以逗號分隔的URI列表，指向API對應路徑。例如：/my-path。hosts，uris或methods至少一個需填才符合規定。
methods
半可選		 指向API的HTTP方法以逗號分隔列表。例如：GET,POST。hosts，uris或methods至少一個需填才符合規定。
upstream_url 指向API服務器的基本目標網址。此URL將用於代理請求。例如：https://example.com。
strip_uri
可選的		 通過其中一個uris前綴匹配API時，從上游URI中刪除匹配的前綴。默認值：true。
preserve_host
可選的		 通過其中一個hosts域名匹配API時，請確保將請求Host頭轉發到上游服務。默認情況下，這是false和上游Host頭將從配置的中提取upstream_url。
retries
可選的		 無法代理時執行的重試次數。默認是5。
upstream_connect_timeout
可選的		 建立與上游服務連接的超時時間（以毫秒為單位）。默認為60000。
upstream_send_timeout
可選的		 用於將請求發送到上游服務的兩個連續寫操作之間的超時（以毫秒為單位）默認為60000。
upstream_read_timeout
可選的		 兩個連續讀取操作之間以超過毫秒的速度發送請求到您的上游服務默認為60000。
https_only
可選的		 如果您希望通過HTTPS僅在適當的端口（8443默認情況下）通過HTTPS提供API來啟用。默認值：false。
http_if_terminated
可選的		 X-Forwarded-Proto在實施僅HTTPS流量時考慮頭。默認值：true。 
curl -i -X POST \
  --url http://localhost:8001/apis/ \
  --data 'name=ptx-APIm-v1' \
  --data 'uris=/MOTC/APIs/v1' \
  --data 'preserve_host=true' \
  --data 'strip_uri= true' \
  --data 'upstream_url=http://ptx.transportdata.backend/MOTC/v1'  

curl -i -X POST \
  --url http://localhost:8001/apis/ \
  --data 'name=ptx-APIm-v2' \
  --data 'uris=/MOTC/APIs/v2' \
  --data 'preserve_host=true' \
  --data 'strip_uri= true' \
  --data 'upstream_url=http://ptx.transportdata.backend/MOTC/v2'

Consumer Object routes

Consumer對象表示API的消費者或用戶。可以依靠Kong作為主資料儲存，也可以將消費者列表與資料庫進行對應，以保持Kong與現有主要儲存之間的一致性。

Endpoint

POST /consumers/
屬性 描述
username
半可選		 consumer的唯一用戶名。請求參數必須至少一個含username 或custom_id。
custom_id
半可選		 用於存儲消費者現有唯一ID的字段 - 用於將Kong與現有資料庫中的用戶進行映射有關。請求參數必須至少一個含username 或custom_id。 
curl -i -X POST \
  --url http://localhost:8001/consumers/ \
  --data "username=Guest" \
  --data "custom_id=MOTC-Guest"

curl -i -X POST \
  --url http://localhost:8001/consumers/ \
  --data "username=General" \
  --data "custom_id=MOTC-General"

Plugin Object routes

  • 詳細請參閱「Kong-Plugin」。

Plugin實體表示將在HTTP請求/響應工作流程期間執行的Plugin配置，運行在Kong之後的API添加功能，例如身份驗證或速率限制。

當在API之上創建添加Plugin時，客戶端發出的每個請求都將通過設置的Plugin的配置進行評估。有時，Plugin需要針對某些特定的消費者調整到不同的值，可以通過指定consumer_id值來實現。

Endpoint

POST /apis/{name or id}/plugins/
屬性 描述
ame or id 添加插件配置的唯一標識符或 API的名稱。如果您正在為每個API添加插件，請使用/plugins端點而不指定api_id。

Request Body

屬性 描述
name 要添加的插件的名稱。目前，插件必須分別安裝在每個Kong實例中。
consumer_id
可選的		 消費者的唯一標識符，覆蓋傳入請求中此特定消費者的現有設置。
config.{property} 插件的配置屬性可以在插件庫中的插件文檔頁面上找到。 
consumers hmac-auth
curl -X POST http://localhost:8001/consumers/Guest/hmac-auth \
    --data "username=Guest0001" \
    --data "secret=GuestSecret001"	

API hmac-auth
curl -X POST http://localhost:8001/apis/ptx-APIm/plugins \
    --data "name=hmac-auth" 

API rate-limiting (config.policy=redis) 需架設redis

curl -X POST http://localhost:8001/apis/ptx-APIm-v2/plugins \
	--data "name=rate-limiting" \
	--data "consumer_id=fbdb9f8a-7026-4445-a083-e2241aa8e8f0" \
	--data "config.second=5" \
	--data "config.limit_by=ip" \
	--data "config.policy=redis" \
	--data "config.redis_host=127.0.0.1" \
	--data "config.redis_database=7" \
	--data "config.redis_timeout=2000" \
	--data "config.redis_port=3389"

Certificate Object routes

證書對象表示SSL證書的公共證書/私鑰對。Kong使用這些對象來處理加密請求的SSL / TLS終止。證書可選地與SNI對象相關聯，以將證書/密鑰對與一個或多個主機名相關聯。

Endpoint

POST /certificates/
屬性 描述
cert PEM編碼的SSL密鑰對的公共證書。
key SSL密鑰對的PEM編碼私鑰。
snis
將此證書與SNI關聯的一個或多個主機名。這是一個糖參數，它將在引擎蓋下創建一個SNI對象，並將其與此證書相關聯，以方便您使用。 
curl -i -X POST \
  --url http://localhost:8001/certificates/ \
  --data 'cert=/usr/local/nginx/ssl/nginx.crt' \
  --data 'key=/usr/local/nginx/ssl/nginx.key' \
  --data 'snis=ssl-example'

SNI Object routes

SNI對象表示主機名與證書的多對一映射。也就是說，證書對象可以有許多與之關聯的主機名; 當Kong收到SSL請求時，它使用客戶端Hello中的SNI字段根據與證書相關聯的SNI來查找證書對象。

Endpoint

POST /snis/
屬性 描述
name ssl-example
ssl_certificate_id ae1be132-a3c9-48e4-a946-78c1e5a628d5 
  curl -i -X POST http://localhost:8001/apis \
    -d "name=ssl-only-api" \
    -d "upstream_url=http://ptx.transportdata.tw/PTX" \
    -d "hosts=ssl-example" \
    -d "https_only=true"

測試

curl -i https://localhost:8443/ \
  -H "Host: ssl-example"

不確定是否自動加入

curl -i -X POST \
--url http://localhost:8001/SNIS/ \
--data 'name=ssl-example' \
--data 'ssl_certificate_id=ae1be132-a3c9-48e4-a946-78c1e5a628d5'

Upstream Object routes

上游對象表示虛擬主機名，可用於通過多個服務（target）對傳入請求進行負載平衡。

Endpoint

POST /upstreams/

Request Body

屬性 描述
name 可以在某個upstream_url字段中引用的api名稱
slots
可選的		 負載平衡器算法中的插槽數（10- 65536，默認為1000）。
orderlist
可選的		 順序但隨機排序的整數數列表，用於確定平衡器中插槽的分佈情況。
如果省略，自動產出。如果給值，必須具有剛好的slots數目個數。		 
    curl -i -X POST \
    --url http://localhost:8001/upstreams/ \
    --data 'name=ptx.transportdata.backend'

Target Object routes

target是具有標識後端服務實例的端口的IP地址/主機名。每個上游都可以有多個target，並且可以動態添加和刪除Target。 由於上游維護Target更改的歷史記錄，因此無法刪除或修改目標。要禁用Target，可設定target weight=0; 或者使用DELETE方法來實現。

Endpoint

POST /upstreams/{name or id}/targets
屬性 描述
name or id 要添加target 唯一識別upstreams的id 或name

Request Body

屬性 描述
target 目標地址（ip或hostname）和端口。如果省略port默認值8000。如果主機名解析為SRV記錄，則該port值將被dns記錄中的值覆蓋。
weight
可選的		 該目標在上游負載平衡器內的權重（0- 1000，默認為100）。如果主機名解析為SRV記錄，則該weight值將被dns記錄中的值覆蓋。 
curl -i -X POST \
--url http://localhost:8001/upstreams/ptx.transportdata.backend/targets/ \
--data 'target=210.242.136.68:80'
